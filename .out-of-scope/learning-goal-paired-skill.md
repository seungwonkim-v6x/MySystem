# Out of scope: DrCatHicks/learning-goal paired skill

**Decided:** 2026-05-17 during v0.33.0 /office-hours + /deep-research.

**Status (2026-07-07): resolved against adoption** — the six-week check never
ran; the learning-opportunities plugin family itself was removed from MySystem
(hook noise, no observed value; see CHANGELOG). Do not adopt learning-goal.

## What it is

[DrCatHicks/learning-goal](https://github.com/DrCatHicks/learning-goal) — the
companion skill to learning-opportunities. Applies Mental Contrasting with
Implementation Intentions (MCII) *before* a task, where learning-opportunities
applies retrieval practice *after*. 170 ⭐, CC-BY-4.0, actively maintained.

## Why it was attractive

Same author, same evidence-based learning-science approach, complementary
phase of work (pre vs post). Easy to bundle.

## Why we rejected it

Premature inclusion = scope creep. learning-opportunities itself is brand new
to the workflow; piling on a second learning skill before observing whether
the first one even gets used is the failure mode the "skill whitelist" rule
is designed to prevent. Six weeks of real-usage data on
learning-opportunities first, then re-evaluate.

## Reconsider when

- Six weeks past 2026-05-17 (≈ 2026-06-28) — check how often
  learning-opportunities lessons were actually accepted vs declined
- If decline rate > 80%, both skills probably aren't fit; don't add learning-goal
- If accept rate is healthy AND a recurring pattern is "I wish I'd named the
  learning goal before starting", adopt
